> 영감: revfactory/fable-5-incident (다중 에이전트 하네스 + 정적 전시 + 출처 추적성·날조 0·정직한 한계)

---

## 1. 정체성 · 원칙

**한 줄 정체성:** "검증된 사실을 전면에 두고 검색하는 정적 플랫폼."
PolitiFact·스놉스 계열에 가깝되, 각 이슈의 여론 지형을 *맥락*으로 함께 보여준다.

fable에서 물려받는 원칙이 여기선 '있으면 좋은 것'이 아니라 **생존 조건**이다:

- **출처 추적성** — 모든 claim·판정·인용은 원본 URL로 추적 가능해야 한다.
- **날조 0** — 검증되지 않은 claim은 발행하지 않는다. 검증된 것만 전시.
- **정직한 한계** — 봇 차단 등으로 수집 못 한 소스는 숨기지 않고 각 이슈의 메서드 섹션에 명시.
- **중립** — 팩트체크는 사실 검증이지 입장 표명이 아니다. 어느 한쪽을 편들지 않는다.
- **데이터 밖 단정 금지** — 검증 범위를 넘는 인과·추정 단정을 하지 않는다.

---

## 2. 합의된 4개 핵심 결정

1. **검색 단위 = 이슈.** 사회 이슈가 카드로 쌓이고, 각 이슈는 하나의 페이지.
2. **추가 방식 = fable식 수동.** 이슈마다 리서치 하네스를 실행 → 정적 페이지 생성 → 커밋.
3. **편집 태도 = 팩트체크 중심.** '무엇이 사실인가'를 전면에, 여론 분포는 맥락으로.
4. **검색 = 키워드 전문검색 + 기간/최신순.** (판정 필터·카테고리 UI는 일부러 뺐고, 스키마만 열어둠.)

**검색 구현 = A안(fable 순정).** 외부 라이브러리 0. vanilla JS 자체 검색.
`index.json` 스키마를 라이브러리 중립적으로 설계해, 검색이 부족해지면 Fuse.js(B안)를 드롭인으로 얹을 수 있게 한다.

## 4. 이슈 페이지 구조 (팩트체크 중심)

순서와 비중이 곧 편집 태도의 표현이다. 팩트체크가 메인, 여론은 맥락.

1. **헤더** — 이슈명 · 발생일/갱신일 · 한 줄 요약
2. **① 핵심 팩트체크 (메인)** — claim별로: 주장 텍스트 → 판정(verdict) → 근거 요약 → 출처 링크
3. **② 타임라인** — 검증된 이벤트의 시간순 정렬
4. **③ 여론 지형 (맥락)** — 찬반·감성 분포, 대표 인용문 (출처 표기)
5. **④ 전체 출처** — 수집·인용된 모든 URL
6. **⑤ 메서드 · 한계** — 수집 기간·대상, 수집 못 한 소스 명시

## 5. `index.json` 스키마 (라이브러리 중립)

배열의 각 원소(= 이슈 1개):

```json
{
  "slug": "example-issue",
  "title": "이슈 제목",
  "summary": "한 줄 요약",
  "category": "사회",                 // 스키마만 존재, UI 필터는 비범위
  "date_occurred": "2026-06-01",
  "date_updated": "2026-06-18",
  "claims": [
    { "text": "검증 대상 주장", "verdict": "대체로 사실" }
  ],
  "quotes": ["대표 인용문 1", "대표 인용문 2"],
  "search_blob": "제목 요약 주장 인용 ... (소문자·공백정규화된 검색용 텍스트)",
  "url": "issues/example-issue/index.html"
}
```

- `search_blob` = 제목 + 요약 + 모든 claim 텍스트 + 모든 인용을 이어붙여 소문자화·공백정규화한 검색용 필드. **한국어는 substring 매칭만으로 충분.** (선택: 나중 랭킹용 bigram 필드를 추가할 여지를 남김.)
- `verdict` 허용값(5단계): `사실` / `대체로 사실` / `논쟁` / `대체로 거짓` / `거짓`.

---

## 6. 검색 동작 (`search.js`)

- 페이지 로드 시 `fetch('index.json')` 1회.
- 입력 키워드를 각 이슈 `search_blob`에 substring 매칭 → 매칭된 이슈 카드만 렌더.
- 기본 정렬: 최신순(`date_updated` 내림차순). 토글로 발생순(`date_occurred`).
- 빈 검색 = 전체를 최신순으로.
- 외부 라이브러리 0개. (B안 전환 시 이 파일만 교체.)

## 8. 검증 방법 (작동 확인 기준)

- 매니페스트 이슈 수 `==` `issues/` 디렉토리 수 (불일치 0).
- 모든 claim에 `verdict` + 출처 URL 존재 (빈 값 0).
- 알려진 키워드로 검색 → 기대한 이슈가 결과에 노출.
- `file://`로 허브를 열어 검색 → 카드 → 이슈 페이지 이동까지 동작.
- `index.json`의 모든 `url`이 실재 파일을 가리킴 (깨진 링크 0).

## 9. 비범위 (YAGNI)

판정 필터·카테고리 UI(스키마만 둠), 댓글/제출/로그인, 서버·DB, 자동 수집 스케줄, 다국어, 검색 라이브러리(B안은 추후 옵션).

- `_workspace/` 를 공개 커밋할지(투명성) vs `.gitignore`할지(노이즈).
